Module Pinnacle

The entry point to configuration.

This module contains the setup function, which is how you'll access all the ways to configure Pinnacle.

pinnacle.setup(config_fn)
Setup a Pinnacle config.

You must pass in a function that takes in the Pinnacle table. This table is how you'll access the other config modules.

You can also require the other modules. Just be sure not to call any of their functions outside this setup function.

If you want to run a function with the config without blocking at the end, see Pinnacle.run.

Parameters:

  • config_fn function

See also:

    Pinnacle.run
pinnacle.run(run_fn)
Run a function with the Pinnacle API.

If you are writing a config, use Pinnacle.setup instead.

Like Pinnacle.setup, this function takes in a function that takes in the Pinnacle table. This allows you to run anything that setup can run.

Unlike setup, this will not listen to the compositor for incoming key presses, signals, and the like. This means that this function will not block and can be used to integrate with external applications like taskbars and widget systems like eww, but it will not allow you to set usable keybinds or call signal callbacks. This is useful for things like querying compositor information for outputs and windows.
